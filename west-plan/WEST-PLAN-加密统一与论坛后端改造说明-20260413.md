# WEST-PLAN 加密统一与论坛后端改造说明（2026-04-13）

## 1. 改造目标

本次改造聚焦两条主线：

1. 统一各前端登录加密算法与后端解密算法，确保协议一致且可平滑兼容历史请求。
2. 聚焦论坛后端模块，补齐核心能力、修复关键逻辑缺陷，并新增可直接用于业务的论坛统计与推荐能力。

---

## 2. 登录加密/解密协议统一

### 2.1 统一后的标准协议

- 前端：`RSA(明文)`，直接提交 RSA 密文字符串（JSEncrypt 输出）。
- 后端：优先按 `RSA 密文` 直接解密。

### 2.2 历史兼容策略

为避免旧客户端瞬时失效，后端在直接 RSA 解密失败后，会自动尝试：

- 兼容旧协议：`Base64(RSA 密文)` -> 先 Base64 解包，再 RSA 解密。

若两种都不匹配，保留原值进入后续流程（维持原系统容错行为）。

### 2.3 已修改前端范围

- forumui
- server-ui
- westplan/ruoyi-ui

三套前端登录均统一为“单层 RSA 密文传输”。

## 3. 论坛后端核心修复与能力增强

### 3.1 已修复问题

1. 帖子详情匿名访问异常
- 修复前：帖子详情依赖强制登录用户，游客访问可能报错。
- 修复后：支持游客查看公开且审核通过帖子，并对非公开/未审核帖子做可见性校验。

2. 最新帖子接口登录依赖问题
- 修复前：接口代码与注释不一致，实际依赖登录上下文。
- 修复后：支持游客调用，登录用户仍可看到自身相关数据。

3. 评论删除逻辑不一致
- 修复前：控制器删除接口绕过了完整权限和计数处理逻辑。
- 修复后：统一走 `deleteForumCommentById`，保证权限校验与评论计数联动正确。

4. 跨帖回复校验缺失
- 修复前：可能出现 parentId 属于其他帖子但仍可提交回复。
- 修复后：新增父评论与帖子一致性校验，防止脏数据。

5. 帖子互动边界不足
- 修复前：对未公开/未通过帖子交互限制不足。
- 修复后：点赞/收藏仅允许公开通过帖子，或作者/管理员操作。

6. 管理参数校验增强
- 帖子置顶/热点接口新增 0/1 参数合法性校验。

### 3.2 新增论坛能力

1. 推荐帖子接口
- 路径：`GET /system/forum_post/recommend`
- 参数：`limit`、`boardId`、`keyword`
- 规则：按热度得分排序（点赞/评论/收藏/浏览综合），支持板块和关键词过滤。

2. 论坛统计概览接口
- 路径：`GET /system/forum_post/stats`
- 返回：帖子总量、已审核/待审核、公开数、今日新增、浏览/点赞/评论/收藏汇总，并附带分板块统计。

3. 帖子最新评论接口
- 路径：`GET /system/forum_comment/recent/{postId}?limit=10`
- 用途：详情页/侧栏最新评论流。
